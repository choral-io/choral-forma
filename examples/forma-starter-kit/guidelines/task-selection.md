# Task Selection

## When To Use This

Use this guideline when you add, review, or reorder tasks in the workspace.

## Guidance

1. Prefer tasks that unblock other work, reduce confusion for readers, or convert repeated discussion into durable knowledge.
2. Keep each task narrow enough to finish in one review cycle. If it needs several independent outcomes, split it.
3. Record task dependencies explicitly with `blocked_by` when the order matters.
4. Move exploratory ideas into a proposal when the team is still deciding what should happen.
5. When replacing starter content with real team content, start with the work that will make [[tasks/add-team-notes|Add Team Notes]] more concrete, such as [[tasks/prepare-first-team-overview|Prepare First Team Overview]].
