# Knowledge Capture

## When To Use This

Use this guideline when you decide a working conversation, draft, or task outcome should become shared knowledge in the workspace.

## Guidance

1. Capture the smallest durable artifact that helps the next reader: a note, a task, a proposal, a decision, or an update to an existing page.
2. Write the shared conclusion, not the entire conversation. Keep temporary exploration in local notes until it becomes durable.
3. Add links that explain why the page matters, following the examples in [[notes/connect-pages|Connect Pages]].
4. When a conclusion changes team behavior, record it as a decision or guideline rather than burying it in a task comment.
5. Revisit link quality after capture so pages such as [[tasks/connect-related-pages|Connect Related Pages]] and [[tasks/collect-onboarding-questions|Collect Onboarding Questions]] stay easy to navigate.
