# Add Weekly Review Rhythm

## Problem

The starter explains how to organize shared knowledge, but it only hints at how a team keeps that knowledge current after copying it into a live workspace.

## Proposed Change

Add a lightweight weekly review habit:

1. review the active tasks in [[tasks/review-starter-workspace|Review Starter Workspace]];
2. decide whether new work belongs in notes, tasks, proposals, or decisions;
3. capture the result using [[guidelines/knowledge-capture|Knowledge Capture]].

## Expected Outcome

The copied workspace stays small, reviewable, and aligned with [[decisions/repository-markdown-is-source-of-truth|Keep Repository Markdown As The Source Of Truth]].
